Firewalls & IDE

  • Method "Fragmentation": The requested scan utilizes tiny fragmented IP packets. The concept involves dividing the TCP header across several packets to complicate detection by packet filters or intrusion detection systems. By default, nmap splits packets into 8 bytes. To divide the packets into 16 bytes, simply add another '-f'. For instance, 'nmap -f -f 192.167.8.3' results in 16 bytes, while 'nmap -f 192.167.8.3' results in 8 bytes. Alternatively, you can use the '--mtu' option to set the size manually, for instance, 'nmap 192.168.8.3 --mtu 32' sets it to 32 bytes. This scan can be accomplished by entering '-f' and requires root privileges to execute.
  • Method "Spoofing - Decoy IP Address": Spoofing means that we attempt to deceive the target system. For example, we could send our scan from different IP addresses separated by commas like this: 'nmap -D 192.167.8.3,192.167.8.6,192.167.8.12,ME TargetIPadress'. ‘ME’ represents the IP address of our Kali Linux system. In the example above, we are sending requests from 4 different IP addresses. The RND function can be used to generate a random, non-reserved IP address. Additionally, appending RND with a colon and a number, such as in 'nmap -D RND:5 192.167.8.3', will produce that quantity of random IP addresses—in this case, five.
  • Method "Spoofing - Hiding own IP Adress" : The '-S' flag in nmap is utilized to set the source IP address for packets that nmap sends during a scan. This can be especially useful for spoofing the source IP address to conceal the identity of the scanning machine or to assess how a system reacts to various IP addresses. But be aware that you won’t receive any results back, because the reply will be sent to the IP address we are trying to impersonate. For example: 'nmap -S 8.8.8.8 192.167.8.3'. But this would not work. We should include the '-Pn' option, which assumes that all hosts are online without discovering any ports. Additionally, we must use the '-e' option to specify a network interface, such as eth0, for instance. We can also add the ‘-g’ option to act like a port that sends a request. For example, ‘nmap -S 8.8.8.8 -e eth0 -g 80 192.167.8.3’.
